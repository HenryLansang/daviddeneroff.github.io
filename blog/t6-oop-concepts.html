Variable Scope

11/16/14


There are many different types of variables in Ruby, so it’s very important to understand what they are and when to use them. In this blog post, we will go over local, global, instance, and class variables in order to get a better idea of what they actually do.

Local variables are contained to where they occur. If you define a local variable inside of a method, it can only be used inside of that method. Local variables have the smallest scope. Below is a method that makes use of a local variable

def example_method
x = 10
puts x
end


Instance variables, marked with a @, work within the same class or inherited class in a program.

class One
def initialize
@x = 12
end
def show
@x
end
end

Class variables, marked with @@, work throughout a given class. The distinction between instance and class variables is that while you could use instance variables throughout a class, using a class variable actually changes the value of the variable in the entire class. The scope of class variables can be tricky, so make sure you understand why you’re using one in order to make sure you get the results that you expect.

class Triangle
@@sides = 3
def self.sides
@@sides
end
end
class Square < Triangle
@@sides = 4
end
puts Triangle.sides # =>4
puts Square.sides # =>4

Global variables can be accessed from anywhere in the entire program, not limited to methods, classes, or anything!! They are marked with a $, so $global would be a global variable. So, their scope covers the entire Ruby program…

$x = 10
