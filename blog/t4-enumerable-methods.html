#map and #map!

11/2/14


In this entry we're going to take a look at both both the destructive and non-destructive versions of the #map method of the enumerable module. The #map method serves great purpose because it takes an enumerable object and a block of code, applies that block to each element of the object, and spits out the value of the object after the block of code has been applied. Check out the example below:
Here we have an example of an array:

[1, 2, 3].map {|x| x**2}
=>[1, 4, 9]

With #map we can look through this array and raise every value to the second power. Afterwards, ruby leaves us with the original array and returns the new array. That is because #map is a non-destructive method. That pretty much just means it won't mess with the original input. But, let's say we wanted to change the input forver, then we'd have to switch to the destructive method #map!. #map and #map! work exactly the same way, the only difference being #map! actually alters the original array. I hope this helped make sense of #map and #map!, happy coding.
